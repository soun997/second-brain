## 성능 목표 잡기

[성능 튜닝 전에 성능 예산부터 구하자](https://velog.io/@sontulip/web-performance-budget)를 참고하여 작성했습니다.

성능 개선은 비용이 드는 작업이다.
그렇기 때문에 서비스의 모든 페이지를 빠르게 만들 수 없다.

예시) 사용자가 '홈 화면'에 접속했을 때 '최대 3초' 안에 콘텐츠를 받아볼 수 있어야 한다.
1. 정적 리소스는 최대 50ms 안에 리턴되어야 한다. (**Web 서버 예산**)
2. API 요청 처리는 DB 조회 시간을 제외하고 최대 100ms을 넘어서는 안된다. (**WAS 예산**)
3. DB에서는 최대 500ms 안에 조회가 이루어져야 한다. (**DB 예산**)

### 연구 결과 참조
1. 로딩 속도가 3초 이상이 될 때 53%의 사용자는 떠나간다.
2. 사용자는 20% 이하의 성능 차이는 인지하지 못한다.
